# 《这个APP能让你戒烟》最终形态完整实现方案

版本：Final Blueprint 1.0
产品类型：个性化 AI 戒烟系统
目标用户：有戒烟意愿、复吸经历、想减少吸烟、或正在考虑戒烟的成年人
产品定位：不靠恐吓、不靠硬忍，而是通过 AI 个性化画像、认知重塑、场景干预、烟瘾急救、复吸修复和价值奖励，帮助用户真正降低对烟的心理依赖，最终完成身份转变：从“我在戒烟”变成“我是不抽烟的人”。

---

# 0. 总体原则

《这个APP能让你戒烟》不是普通戒烟打卡 APP，不是健康科普 APP，也不是单纯 AI 聊天机器人。

它的终局形态是：

> 一个基于用户画像、行为数据、AI 多智能体、场景预测、个性化内容和即时干预的戒烟操作系统。

产品核心不是告诉用户“吸烟有害健康”，而是解决以下问题：

1. 用户为什么抽烟？
2. 用户什么时候最容易抽？
3. 用户以为烟给了他什么？
4. 用户真正想要的东西是什么？
5. 如何在用户想抽的那几分钟及时介入？
6. 如何把“不抽烟”转化成即时可见收益？
7. 如何在复吸后不让用户彻底放弃？
8. 如何让用户最终不再觉得自己在“戒”，而是已经“不需要烟”？

---

# 1. 证据与产品边界

本产品是行为支持和数字化戒烟辅助工具，不应宣称“保证戒烟成功”，也不应替代医生、戒烟门诊或药物治疗。

国际权威戒烟建议通常强调行为支持、戒烟计划、应对烟瘾和压力、必要时结合药物或尼古丁替代治疗。CDC 明确提到，戒烟咨询可以帮助用户制定戒烟计划，并准备应对压力、烟瘾和戒烟过程中的其他问题。([疾病控制与预防中心][1]) WHO 2024 年成人烟草戒断临床治疗指南也建议，将行为干预和药物治疗结合可显著提升戒烟成功机会，并列出尼古丁替代治疗、伐尼克兰、安非他酮和金雀花碱等有效治疗方式。([世界卫生组织][2])

数字化戒烟干预可以作为独立自助服务，但 WHO 指南也强调，当数字干预作为独立服务时，应告知用户还有其他推荐的戒烟干预方式及其有效性。([国家生物技术信息中心][3]) 因此，本产品必须在适当位置提供“专业支持入口”，例如戒烟门诊、热线、药物/NRT 信息、医生咨询建议等。

移动端戒烟干预有一定证据基础，但不能过度夸大。Cochrane 综述显示，自动化短信类戒烟干预相较最低支持可以提高戒烟率，证据确定性为中等；但智能手机 APP 干预的效果证据整体更复杂，2023 年 JMIR 系统综述指出 APP 戒烟干预证据仍需进一步明确。([PubMed][4]) 因此，本产品的设计策略应是：吸收短信干预的“及时、个性化、持续支持”优势，同时用 AI 和行为数据增强 APP 的场景化干预能力。

# 2. 产品一句话定义

**《这个APP能让你戒烟》是一款 AI 个性化戒烟系统，通过理解用户为什么抽烟、预测用户什么时候想抽、在关键场景及时介入、重塑用户对烟的认知，并把不抽烟转化成金钱、家庭、自由和身份收益，帮助用户从“忍住不抽”走向“我不需要烟”。**

# 3. 产品核心理念

## 3.1 不做健康恐吓

健康风险可以作为信息存在，但不作为主叙事。

很多用户不是不知道吸烟有害，而是明知道有害仍然抽。因此产品主线不能是：

> 抽烟会伤身体，你要自律。

而应该是：

> 烟没有给你真正的快乐。它只是制造缺口，再假装填补缺口。

---

## 3.2 不靠意志力硬扛

产品不要让用户长期处于“我很想抽，但我要忍住”的状态。

核心策略是：

> 让用户越来越不相信“这支烟真的有价值”。

---

## 3.3 不做单一人群产品

不是只服务有宝宝的人，不是只服务上班族，也不是只服务金钱敏感用户。

产品采用：

> 固定产品骨架 + 个性化模块组合 + AI 动态话术。

所有用户使用同一套底层系统，但每个人看到的首页、提醒、内容、奖励、AI 话术和急救流程不同。

---

## 3.4 不羞辱复吸

复吸不是归零，不是失败，不是人格问题。

产品必须把复吸定义为：

> 一次高危场景数据采集。

复吸后产品要做的是修复，而不是惩罚。

---

## 3.5 最终目标是身份转变

最终用户不应该觉得：

> 我很能忍，我戒了很久。

而应该觉得：

> 我是不抽烟的人。烟已经不是我的生活选项。

# 5. 用户完整旅程

## 5.1 第一次打开 APP

用户进入后，不要直接注册、打卡、问“你今天抽了几支”。

应进入一个有仪式感的流程：

> 我们先不要求你立刻戒烟。
> 我们先理解：烟在你的生活里到底扮演了什么角色。

流程：

1. 欢迎页
2. 产品理念页
3. 戒烟心理扫描
4. 生成戒烟画像
5. 选择自由日或观察模式
6. 进入个性化首页

---

## 5.2 第一周

目标不是让用户“坚持 7 天”，而是完成三个动作：

1. 识别自己的高危场景
2. 建立“不抽烟也能休息/放松/奖励”的替代路径
3. 每次想抽时使用“我想抽了”急救系统

---

## 5.3 第一个月

目标是完成认知转变：

1. 烟不是放松
2. 烟不是奖励
3. 烟不是自由
4. 烟不是社交能力
5. 烟不是人生乐趣
6. 不抽烟不是损失

---

## 5.4 长期阶段

目标是身份巩固：

1. 用户不再高频打开 APP
2. 用户只在高危场景使用 APP
3. 用户把自己定义为“不抽烟的人”
4. APP 从急救工具转为长期守护系统

# 6. 戒烟画像系统

戒烟画像是整个产品的底层中枢。

问卷不是为了收集资料，而是为了生成可用于功能分发、内容推荐、AI 话术和干预策略的用户模型。

---

## 6.1 画像维度

### 维度一：基础信息

字段：

* 年龄
* 性别
* 所在地区
* 职业
* 工作形态：办公室 / 户外 / 自由职业 / 学生 / 销售 / 司机 / 其他
* 作息规律：规律 / 经常熬夜 / 经常加班 / 轮班
* 居住状态：独居 / 与伴侣同住 / 与父母同住 / 有孩子 / 集体宿舍

---

### 维度二：吸烟历史

字段：

* 烟龄
* 一天平均支数
* 一包烟价格
* 一包烟通常抽几天
* 常抽品牌
* 每天第一支烟时间
* 是否夜间抽烟
* 是否曾经戒烟
* 最长戒烟多久
* 为什么复吸
* 是否使用过戒烟药物/NRT
* 是否去过戒烟门诊

---

### 维度三：吸烟触发场景

多选，但后续会转化为分数。

场景：

* 起床后
* 上厕所
* 吃饭后
* 工作累了
* 写代码/办公间隙
* 开完会
* 下午犯困
* 压力大
* 情绪烦躁
* 无聊
* 打游戏
* 刷短视频
* 喝咖啡
* 喝酒
* 开车
* 等人
* 同事递烟
* 朋友聚会
* 睡觉前
* 回家前
* 其他

---

### 维度四：戒烟动机

动机：

* 健康
* 省钱
* 宝宝
* 伴侣
* 父母
* 形象
* 口气/烟味
* 工作效率
* 自由感
* 自尊
* 不想偷偷摸摸
* 不想被烟控制
* 想证明自己可以
* 其他

用户不仅选择，还要排序：

> 哪三个原因最能让你真的停下来？

---

### 维度五：心理信念

要识别用户对烟的幻想。

问题示例：

1. 你觉得烟能让你放松吗？
2. 你觉得饭后一支烟很舒服吗？
3. 你觉得工作累了抽一支能恢复状态吗？
4. 你觉得抽烟是奖励自己吗？
5. 你觉得不抽烟人生会少点意思吗？
6. 你觉得社交场合不抽烟会尴尬吗？
7. 你觉得戒烟主要靠意志力吗？
8. 你觉得你是戒不掉，还是暂时没找到方法？

输出信念标签：

* 放松幻觉
* 奖励幻觉
* 社交幻觉
* 灵感幻觉
* 无聊补偿
* 反叛/不良嗜好认同
* 身份绑定
* 低自信戒烟预期

---

### 维度六：依赖程度

可参考简化尼古丁依赖评估，但产品内不一定使用医学量表名称，避免用户产生被诊断感。

问题：

* 醒来多久会抽第一支？
* 禁烟场所是否很难忍？
* 生病时是否还想抽？
* 一天不抽是否明显烦躁？
* 有没有为了抽烟专门离开当前场景？
* 有没有夜里醒来想抽？

输出：

* 低依赖
* 中依赖
* 高依赖
* 建议专业支持

高依赖用户应提示：

> 你可能不只需要行为工具，也可以考虑专业戒烟支持、医生建议或合规戒烟药物信息。

---

### 维度七：支持环境

字段：

* 家人是否支持戒烟
* 伴侣是否反感烟味
* 是否有孩子
* 孩子年龄
* 是否愿意加入家庭支持
* 同事是否经常抽烟
* 工作地点是否容易接近吸烟区
* 朋友是否经常递烟
* 是否愿意公开戒烟
* 是否想隐私模式

---

### 维度八：戒烟阶段

状态机：

| 状态                | 含义            |
| ----------------- | ------------- |
| OBSERVING         | 观察期，还没决定戒     |
| PREPARING         | 准备期，想戒但未开始    |
| QUIT_DAY          | 自由日，当天开始不抽    |
| EARLY_QUIT        | 戒烟第 1-7 天     |
| STABILIZING       | 戒烟第 8-30 天    |
| IDENTITY_BUILDING | 30 天以上，身份巩固   |
| LAPSE_REPAIR      | 抽了一支或少量复吸     |
| RELAPSE_RECOVERY  | 明显恢复吸烟，需要重建计划 |

每个用户维护以下权重：

```json
{
  "motivation_weights": {
    "family": 0.95,
    "money": 0.85,
    "freedom": 0.75,
    "health": 0.30,
    "image": 0.55,

  }

```

---

## 7.2 权重更新依据

用户行为会影响权重：

| 行为          | 系统解释       |
| ----------- | ---------- |
| 经常点击宝宝卡片    | 家庭动机权重上升   |
| 经常查看烟钱银行    | 金钱动机权重上升   |
| 健康科普跳出率高    | 健康内容权重下降   |
| 下午频繁点“我想抽了” | 下午高危权重上升   |
| 工作场景复吸      | 工作触发权重上升   |
| 社交场景复吸      | 社交触发权重上升   |
| 完成 90 秒重置   | 该干预策略有效性上升 |
| 跳过长文        | 长文内容权重下降   |

## 7.3 首页个性化规则

首页采用槽位制。

固定槽位：

1. 身份槽
2. 主动力槽
3. 高危预警槽
4. 急救入口槽
5. 收益槽
6. 今日任务槽

槽位内容由权重决定。

### 示例：家庭 + 金钱 + 工作触发用户

首页显示：

* 你已经自由了 3 天 12 小时
* 今天你没有把烟味带回家
* 宝宝无烟账户 +72 元
* 下午 3 点是高危时间，不要下楼
* 今日任务：工作累了也不走向吸烟区

### 示例：金钱 + 社交用户

首页显示：

* 你已经自由了 3 天 12 小时
* 已省下 72 元
* 距离目标奖励还差 428 元
* 晚上聚会高危，提前开启社交防护
* 今日拒烟话术：最近不抽了，状态好多了

### 示例：自由 + 无聊用户

首页显示：

* 你已经 3 天没有被烟安排
* 今天击退烟瘾 4 次
* 无聊不是烟的理由
* 今日小爽点：用 15 分钟换一个无烟奖励
* 我想抽了按钮

# 8.6 烟钱银行系统

烟钱银行不是简单计算省了多少钱，而是把“不抽烟”转化为真实价值。

---

## 计算逻辑

用户输入：

* 一包烟价格
* 一包几支，默认 20
* 一天平均几支
* 当前实际抽烟数量

计算：

```text
单支价格 = 一包价格 / 20
每日预期烟钱 = 单支价格 * 历史日均支数
已省金额 = 单支价格 * 少抽支数
年度预测 = 每日预期烟钱 * 365
```

---

## 账户类型

* 自我奖励账户
* 宝宝账户
* 伴侣礼物账户
* 家庭旅行账户
* 数码产品账户
* 游戏账户
* 存钱账户
* 自定义账户

---

## 页面展示

示例：

> 你已经少抽 156 支。
> 已省下 312 元。
> 这不是小钱，这是你本来打算烧掉的钱。

如果用户有宝宝：

> 宝宝无烟账户：312 元。
> 这笔钱没有变成烟灰，而是留在了家里。

如果用户设置奖励：

> 距离你的机械键盘目标还差 288 元。
> 按当前速度，12 天后可以达成。

# 8.9 复吸无羞辱修复系统

这是产品留存和真实有效性的核心。

---

## 用户记录复吸时

按钮：

> 我抽了一支

弹出文案：

> 好，我们不评价这支烟。
> 我们只看它是怎么发生的。

---

## 复盘问题

1. 什么时候抽的？
2. 在哪里？
3. 和谁？
4. 抽之前是什么感觉？
5. 是自己买的，还是别人给的？
6. 抽之前有没有犹豫？
7. 抽完后真的舒服了吗？
8. 如果重来一次，哪个环节最容易改？

---

## AI 输出

示例：

> 这次不是因为你真的需要烟。
> 是因为“午饭后 + 同事在场 + 无聊聊天 + 手里刚好有烟”同时出现。
> 下次只要提前改变其中一个条件，风险就会明显下降。

然后给出预案：

> 下次午饭后，不跟同事走向吸烟区。
> 先去买水。
> 如果 3 分钟后还想抽，再点“我想抽了”。

---

## 复吸后的状态处理

禁止归零。

错误：

> 你的戒烟记录已重置。

正确：

> 你的自由记录没有消失。
> 这只是一个高危场景被发现了。

但数据上要区分：

* 连续无烟天数
* 总无烟天数
* 复吸事件
* 回归速度

输入：

## 9.6 安全边界 Agent

所有 AI 输出必须通过安全边界检查。

禁止内容：

* 保证戒烟成功
* 替代医生诊断
* 推荐具体处方药用法
* 羞辱用户
* 道德绑架
* 诱发焦虑
* 暗示用户复吸后完蛋
* 对孕妇、青少年、严重依赖用户给出不负责任建议

需要触发专业支持提示的情况：

* 高依赖
* 长期严重戒断反应
* 情绪崩溃
* 明显抑郁/自伤表达
* 怀孕或备孕
* 合并酒精/药物依赖
* 用户询问戒烟药物具体剂量

def recommend_modules(profile):
    modules = ["craving_rescue", "relapse_repair", "identity_system"]

    if profile.motivation.family > 0.7:
        modules.append("family_baby_account")

    if profile.motivation.money > 0.5:

    if profile.triggers.work_break > 0.6:

    if profile.triggers.social > 0.6:
        modules.append("social_refusal_mode")

    if profile.triggers.boredom > 0.6:
        modules.append("boredom_replacement_mode")

    if profile.triggers.after_meal > 0.6:
        modules.append("after_meal_mode")

    if profile.dependence_level in ["high", "very_high"]:

    return modules

    if trigger == "work_break":
        return "work_reset_90s"

    if trigger == "after_meal":
        return "after_meal_3min"

    if trigger == "social":
        return "social_refusal_script"

# 21. 产品最终形态总结

《这个APP能让你戒烟》的终极价值不是记录用户戒了多少天，而是完成一整套闭环：

> 认识我 → 预测我 → 提前保护我 → 当场接管我 → 奖励我 → 复吸后拉回我 → 重塑我。

> 这个 APP 不是在监督我。

这就是《这个APP能让你戒烟》的最终施工蓝图。
